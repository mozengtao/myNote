# sed 匹配引用技巧总结：& 与 \1, \2...

在 `sed` 的 `s`（替换）命令中，引用匹配到的内容是实现自动化文本修改的核心技术。

---

## 1. 全匹配引用：`&`
`&` 代表正则表达式所匹配到的**整个字符串**。

### 核心特点
- **无需分组**：不需要在模式中使用括号。
- **整体性**：它总是代表匹配出的完整部分，不能拆分。
- **场景**：适用于给原有内容添加前缀、后缀或包裹符号。

### 典型示例
| 需求 | 命令示例 | 结果说明 |
| :--- | :--- | :--- |
| **添加注释** | `sed 's/old_cmd/# &/'` | `old_cmd` -> `# old_cmd` |
| **加引号/括号** | `sed 's/[0-9]*/(&)/'` | `123` -> `(123)` |
| **单位转换/标记** | `sed 's/ERROR/[&]/'` | `ERROR` -> `[ERROR]` |

---

## 2. 分组引用：`\1`, `\2`, ...
通过在正则表达式中使用 `\(` `\)`（或扩展正则中的 `()`）将匹配内容划分为多个小组，按顺序编号调用。

### 核心特点
- **灵活性**：可以精准提取或重新排列字符串的一部分。
- **多组支持**：通常支持 `\1` 到 `\9`。
- **场景**：适用于提取关键信息、交换字段顺序、格式化复杂日志。

### 典型示例
| 需求 | 命令示例 (使用 -E) | 结果说明 |
| :--- | :--- | :--- |
| **交换位置** | `sed -E 's/(A):(B)/\2:\1/'` | `apple:orange` -> `orange:apple` |
| **数据提取** | `sed -E 's/.*ID=([0-9]+).*/\1/'` | 从长行中只提取 ID 数字 |
| **清理空白** | `sed -E 's/ +([0-9]+)/\1/'` | 去除数字前的多余空格 |

---

## 3. 快速对比

| 维度 | `&` (Whole Match) | `\1, \2...` (Capture Groups) |
| :--- | :--- | :--- |
| **定义方式** | 自动生成，匹配即存在 | 必须手动用 `()` 包裹内容 |
| **主要用途** | 在原内容基础上装饰（加料） | 提取、修剪或重新排列内容 |
| **代码可读性** | 极简，一眼看出是全引用 | 逻辑较强，需注意括号嵌套顺序 |

---

## 4. 综合实战案例

**需求**：将配置文件中的 `PATH=/usr/bin` 修改为注释掉原行，并在下方新增一行改为 `/usr/local/bin`。

**命令**：
```bash
# 结合了 & (保留原行注释) 和 \1 (提取变量名)
sed -E 's/(PATH)=.*/# &\n\1=\/usr\/local\/bin/' config.txt
```
