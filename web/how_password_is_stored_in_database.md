# 数据库密码安全存储与验证流程

## 一、核心原则
- **禁止明文存储**：密码绝不能以原始文本形式存入数据库，否则数据库泄露会直接导致所有密码暴露。
- **单向哈希 + 加盐**：采用现代密码哈希算法，并为每个密码生成唯一随机盐（Salt），抵御彩虹表、暴力破解等攻击。

---

## 二、密码存储流程
1.  **生成随机盐**：为每个用户密码生成一个**唯一的随机字符串**作为盐（Salt）。
2.  **组合密码与盐**：将用户输入的原始密码与生成的盐进行拼接（如 `password + salt`）。
3.  **计算哈希值**：使用现代安全哈希算法（如 bcrypt、Argon2、PBKDF2 等，避免 MD5、SHA-1 等不安全算法），对拼接后的字符串进行哈希计算。
4.  **存储数据**：将计算得到的**哈希值（Hash）**和对应的**盐（Salt）**一同存入数据库（盐无需加密，可明文存储）。

---

## 三、密码验证流程（用户登录时）
1.  **获取盐与哈希**：根据用户标识从数据库中取出该用户对应的盐（Salt）和存储的哈希值（Stored Hash）。
2.  **重新计算哈希**：将用户本次输入的密码与取出的盐按相同规则拼接，使用**相同的哈希算法**计算新的哈希值（Computed Hash）。
3.  **比对哈希值**：将新计算的哈希值与数据库中存储的哈希值进行比对：
    - 若两者完全一致 → 密码验证通过，允许登录。
    - 若不一致 → 密码验证失败，拒绝登录。

---

## 四、关键安全要点
- **使用现代慢哈希算法**：选择设计为“计算缓慢、消耗资源”的算法，大幅增加暴力破解成本。
- **盐的唯一性**：每个密码必须使用独立的随机盐，避免相同密码产生相同哈希，防止彩虹表攻击。
- **盐无需保密**：盐的作用是保证哈希唯一性，并非密钥，因此可以明文存储在数据库中。
