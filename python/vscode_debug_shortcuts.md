# VSCode Python Debug 核心快捷键速查表
## 一、调试全局启停
| 快捷键 | 功能说明 |
|--------|----------|
| F5 | 启动调试 / 继续执行，运行至下一个断点 |
| Shift + F5 | 终止当前调试会话 |
| Ctrl + Shift + F5 | 重启调试，从头重新运行程序 |

## 二、单步调试核心（最常用）
| 快捷键 | 功能 | 适用场景 |
|--------|------|----------|
| F10 | 单步跳过 Step Over | 执行当前整行代码，跳到下一行；**不进入自定义函数/生成器内部** |
| F11 | 单步步入 Step Into | 若当前行包含函数调用，**钻进函数/生成器内部逐行调试** |
| Shift + F11 | 单步跳出 Step Out | 直接跑完当前整个函数，跳出回到上层调用处 |

## 三、断点基础操作
| 操作 | 说明 |
|------|------|
| 鼠标单击行号左侧 | 切换普通断点（红点=启用，空=取消） |
| 右键行号 → 添加条件断点 | 仅满足自定义表达式条件时才暂停 |
| Ctrl + Shift + F9 | 删除当前行断点 |

## 四、调试侧边栏与面板
| 快捷键 | 功能 |
|--------|------|
| Ctrl + Shift + D | 打开「运行和调试」面板 |
| Ctrl + Shift + Y | 调出内置终端（配合 `console: integratedTerminal` 支持input输入） |

## 五、监视与变量查看（手动操作）
1. 左侧「监视(WATCH)」面板点 +，输入表达式可实时查看变量/`g.gi_frame.f_lasti` 等底层属性
2. 点击调用堆栈里任意栈帧，变量面板会切换为该帧的局部上下文

## 六、生成器/Yield 调试专属操作建议
1. 进入 `next(g)` 调用行时使用 **F11** 步入生成器帧
2. yield 暂停后 F5 放行，生成器栈帧脱离主线调用栈但仍保存在 `gi_frame`
3. 监视面板固定添加表达式快速观察上下文：
```
g
g.gi_frame
g.gi_frame.f_locals
g.gi_frame.f_lasti
```
