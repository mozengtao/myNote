## AWK 系统化掌握手册（中文 · 工程视角）

面向目标：把 AWK 当成**流式（streaming）文本处理引擎/DSL**来理解与使用，而不是“若干记忆命令的集合”。你最终要形成的是：**输入模型 + 执行生命周期 + 语法规则 + 常见模式 + 调试方法**这套可复用心智模型。

---

## Part 1 — AWK 执行模型（核心架构）

### 1) 输入模型：逐条记录（record）→ 字段（field）拆分

- **AWK 是流式处理**：输入按顺序读入，不需要把全文件读到内存（除非你自己把它存进数组）。
- **默认处理单位**是“记录（record）”，通常就是“一行”。这由 `RS`（Record Separator）决定。
- 每读入一条记录，AWK 会把它存到 `$0`，然后按 `FS`（Field Separator）把 `$0` 拆成 `$1..$NF`：
  - `FS` 默认是**空白分隔**（连续空格/Tab 会被当成一个分隔，且会自动忽略行首尾空白，这点非常重要）。
  - 输出时字段之间用 `OFS` 连接，记录末尾用 `ORS` 结束。

你可以把它想成每条记录都会经历一次“自动词法切分（field splitting）”的流水线。

### 2) 执行流水线（非常重要）：BEGIN → 主循环 → END

AWK 程序由若干 `pattern { action }` 规则组成。对每条输入记录，AWK 会按规则顺序做：

```text
[BEGIN]
  ↓
[Read Line]      读入一条记录（通常一行），放入 $0
  ↓
[Pattern Match]  逐条规则判断 pattern 是否为真
  ↓
[Action Block]   对匹配的规则执行 { ... }
  ↓
 (loop)          下一条记录
  ↓
[END]
```

更“执行模型”一点的描述：

- **BEGIN 阶段**：在读任何输入之前执行 `BEGIN { ... }`（做初始化、设分隔符、打印表头、预加载配置等）。
- **主循环阶段**：每读一条记录，依次评估每条规则的 pattern：
  - pattern 为真 → 执行对应 action（动作块）。
  - pattern 为假 → 跳过该规则。
- **END 阶段**：所有输入处理完（或 `exit`）后执行 `END { ... }`（做汇总、收尾输出）。

### 3) 关键内建变量（你必须形成“随手就能用”的肌肉记忆）

- **`NR`**：已处理的记录总数（跨所有文件累计）。
- **`FNR`**：当前文件内已处理的记录数（每换一个输入文件就从 1 重新计数）。
- **`NF`**：当前记录的字段数。
- **`$0`**：当前记录原始文本（整行/整条记录）。
- **`$1..$n`**：第 1..n 个字段；`$NF` 是最后一个字段。
- **`FS` / `RS`**：输入字段/记录分隔符。
- **`OFS` / `ORS`**：输出字段/记录分隔符。

最小高杠杆自检（看懂就等于把主循环抓住了）：

```bash
awk '{print "NR=" NR, "FNR=" FNR, "NF=" NF, "first=" $1, "last=" $NF, "raw=" $0}' file
```

## Part 2 — AWK 完整语法骨架（你写程序的“外壳”）

### 1) 单条规则形式

```text
awk 'pattern { action }' file
```

AWK 程序本质上是“很多条规则的集合”，每条规则是：

```text
pattern { statements }
```

### 2) 多行程序形式（推荐写复杂逻辑时用）

```text
awk '
BEGIN { ... }
pattern { ... }
pattern { ... }
END { ... }
' file
```

工程上更稳定的方式（避免引号地狱）是把脚本放到文件里：

```bash
awk -f script.awk file
```

### 3) Pattern（模式）类型：你在写“什么时候触发”

AWK 的强大之处在于 pattern 不只是正则，它是一个**条件表达式**系统。

- **正则匹配 pattern**：`/regex/`
  - 为真表示当前记录 `$0` 能匹配该正则。
- **表达式 pattern**：`NR == 1`、`$1 == "foo"`、`$3 > 100`
  - 为真就触发。
- **范围（range）pattern**：`/start/, /end/`
  - 从匹配 `start` 的那条记录开始一直为真，直到匹配 `end` 的那条记录（包含两端）。

范围 pattern 的工程直觉：它是一个内建“开关”，开了就持续匹配，直到关。

## Part 3 — Action Block `{}` 深入（关键到决定你是否真正会写 awk）

### 1) `{}` 的本质

- `{ ... }` 是一个**语句块（block）**：当 pattern 为真时执行里面的语句。
- **pattern 省略**：表示对每条记录都执行（等价于 pattern 恒真）。

例：对每行原样输出（显式写法）：

```bash
awk '{ print $0 }' file
```

### 2) 语句分隔规则（Statement Separation Rules）

AWK 语句需要被分隔。合法分隔符是：

#### (1) 换行（newline）

```awk
{
x = 1
y = 2
print x + y
}
```

#### (2) 分号 `;`

```awk
{ x = 1; y = 2; print x + y }
```

#### (3) 混用（最常见）

```awk
{
x = 1; y = 2
print x + y
}
```

关键澄清（工程上要记住“为什么”）：

- **大多数情况下：换行 ≈ 分号**。AWK 的语句边界通常可由换行推断。
- **一行脚本**里你没有换行，语句必须靠 `;` 分隔，因此 one-liner 常见 `;`。
- 但也要知道：某些续行/表达式场景里，换行并不等价于分号（例如你刻意让表达式跨行、或在括号/逗号上下文里换行）。因此当你写复杂脚本时，推荐**清晰的换行+缩进**，减少歧义。

### 3) 隐式默认行为（非常实用）

- **缺省 action**：如果只有 pattern 没有 `{}`，默认动作是 `{ print $0 }`。
  - 例：只写条件，相当于“满足条件就打印整行”：

```bash
awk '$3 > 100' file
```

- **缺省 pattern**：如果只有 `{}` 没有 pattern，表示匹配所有行。

```bash
awk '{print}' file
```

这两条默认行为解释了 AWK 为什么能写出极短但很强的 one-liner：你省掉的并不是能力，而是语法噪音。

## Part 4 — 控制结构（Control Structures）

AWK 的控制结构和 C 很像，但要从“流式每记录执行”的角度理解：**控制结构控制的是当前记录上的 action 执行路径**。

### 1) `if / else`

```awk
if (condition) {
  ...
} else {
  ...
}
```

工程要点：把 pattern 用来做“粗过滤”，把 if 用来做“细分支”，脚本更清晰。

### 2) `while`

```awk
while (condition) {
  ...
}
```

### 3) `do ... while`

```awk
do {
  ...
} while (condition)
```

### 4) `for`

#### C 风格

```awk
for (i = 1; i <= 10; i++) {
  ...
}
```

#### 遍历关联数组（Associative Array）

```awk
for (k in arr) {
  print k, arr[k]
}
```

工程要点：`for (k in arr)` 的遍历顺序**通常不保证稳定**（不同 awk 实现可能不同）。如果你需要排序输出，必须显式排序（gawk 有 `asorti/asort`，POSIX awk 需要自己实现或借助外部 `sort`）。

### 5) `break / continue`

- **`break`**：跳出当前最近的一层循环（`for/while/do-while`）。
- **`continue`**：结束当前迭代，进入下一次循环迭代。

### 6) `next / exit`（非常重要：它们改变“主循环”的控制流）

- **`next`**：立刻停止处理当前记录，直接读下一条记录（回到“Read Line”阶段）。
- **`exit`**：立刻终止整个 awk 程序（随后仍会执行 `END` 块，如果存在）。

对比示例（看差异的最佳方式是写出“会打印什么”）：

```bash
# next：跳过当前行剩余规则/语句，但程序继续处理后面的行
awk '
  /SKIP/ { print "hit skip at NR=" NR; next }
  { print "keep:", $0 }
  END { print "END reached" }
' file

# exit：提前结束整个程序（END 仍会执行）
awk '
  /STOP/ { print "hit stop at NR=" NR; exit }
  { print "keep:", $0 }
  END { print "END reached" }
' file
```

为什么要有 `next`：它让你写出清晰的“护栏式代码”（先处理特殊情况并跳过），减少深层嵌套。

## Part 5 — 函数（Functions）

### 1) 常用内建函数（Built-ins）

#### 字符串

- `length(s)`：长度（省略参数时默认 `length($0)`）
- `substr(s, start[, len])`：子串（start 从 1 开始）
- `index(s, t)`：子串位置（找不到返回 0）
- `split(s, arr[, sep])`：按 sep 拆分到数组（返回字段数）

最小例子：

```bash
awk '{ print length($0) }' file
awk '{ print substr($1, 1, 3) }' file
```

#### 数学

- `int(x)`：向 0 截断（不是 floor，负数要小心）
- `rand()`：0 到 1 的伪随机数（可配合 `srand()` 初始化）

#### I/O（语句/机制）

- `print`：按 `OFS/ORS` 输出
- `printf`：C 风格格式化输出（**不会自动追加 ORS**，你自己加 `\n`）

工程要点：当你需要对齐列、控制小数位、输出固定格式日志时，用 `printf`。

### 2) 用户自定义函数

```awk
function add(a, b) {
  return a + b
}
```

关键解释（为什么重要）：

- **参数传递**：标量参数是“值语义”（你在函数里改 `a` 不会影响外面变量，除非你传的是数组或用了全局变量）。
- **数组参数**：数组是“引用语义”（传进去后在函数内修改会影响外部）。
- **返回值**：用 `return expr` 返回；不写 return 通常等价于返回空/0（依实现细节，工程上不要依赖隐式返回）。

## Part 6 — 数组（AWK 的关键特性：关联数组）

### 1) 关联数组 = 哈希表（hash map）

- 下标可以是字符串（AWK 会把下标当字符串键）。
- 无固定大小，按需增长。

经典计数器：

```bash
awk '
  { count[$1]++ }
  END {
    for (k in count)
      print k, count[k]
  }
' file
```

为什么这很强：它让你在**一次线性扫描**里做聚合统计（频次、分组求和、去重等），不需要数据库也不需要把全文件排序。

## Part 7 — 真实世界常用模式（最重要：把语法变成“能干活”）

### 1) 列提取（column extraction）

```bash
awk '{print $1, $3}' file
```

为什么：AWK 默认就是“按列理解文本”，这比 `cut` 更灵活（能做条件、计算、格式化）。

### 2) 过滤（filter lines）

```bash
awk '$3 > 100' file
```

为什么：pattern 直接就是条件表达式，省掉显式 `{print}`。

### 3) 日志分析（regex filter）

```bash
awk '/ERROR/ {print $0}' file
```

为什么：`/regex/` 是最自然的“流式告警探针”写法。

### 4) 聚合（aggregation）

```bash
awk '{sum += $1} END {print sum}' file
```

为什么：把每行的贡献累加到状态变量，最后在 END 输出汇总，这是最典型的“流式计算”模式。

## Part 8 — 调试技巧（Debugging）

AWK 的调试核心原则：**把你脑中的执行模型外显化**（打印 NR/FNR/NF、关键字段、分支命中）。

### 1) Print 调试

```bash
awk '{ print "DEBUG:", $0 }' file
```

### 2) 跟踪变量/字段

```bash
awk '{ print "NR=" NR, "NF=" NF, "$1=" $1 }' file
```

### 3) 用 BEGIN 做“环境设定/开关”

典型用途：

- 在 `BEGIN` 里设 `FS/OFS/RS/ORS`
- 在 `BEGIN` 里打印表头或初始化累加器
- 用 `-v debug=1` + `BEGIN{...}` 做调试开关

示意：

```bash
awk -v debug=1 '
  BEGIN { OFS="\t" }
  {
    if (debug) print "DBG", NR, NF, $1 > "/dev/stderr"
    print $1, $3
  }
' file
```

为什么把调试信息打到 stderr：不污染主输出（主输出可以继续管道到 `sort`/重定向到文件）。

### 4) 用 END 做“汇总/一致性检查”

```bash
awk '
  { bad += ($3 == "") }
  END { print "bad_rows=" bad }
' file
```

## Part 9 — 高阶洞见（工程视角）

### 1) AWK 是流式计算 DSL

它的抽象不是“文件编辑”，而是“对每条记录执行一段小程序”，并且天然支持：

- 状态变量（跨记录累积）
- 关联数组（分组/去重/聚合）
- 正则（模式识别）

这三者组合，使它非常适合一次性、可复现、低依赖的数据处理任务。

### 2) 为什么 AWK 特别适合

- **日志处理**：输入是行流；你要的往往是筛选、提取字段、统计频次、按 key 聚合。
- **数据变换**：把“半结构化文本”转成“列化输出”（TSV/CSV/对齐文本）。

### 3) 与 grep / sed 的工程对比（为什么要选对工具）

- **grep**：强在“过滤/定位”——基本是 pattern → 输出匹配行（或匹配片段）。
- **sed**：强在“替换/编辑流”——更像逐行应用编辑命令（替换、插入、删除）。
- **awk**：强在“计算与结构化”——不仅能过滤，还能按列计算、聚合、格式化输出。

一句话决策：

- 只是找行：先用 `grep`。
- 主要是替换：用 `sed`。
- 需要按字段计算、统计、格式化、分组：用 `awk`。

## 附：一组“最小但强力”的常用命令行姿势（建议你背下来）

```bash
# 指定字段分隔符（最常用）
awk -F',' '{print $1, $3}' file.csv

# 通过 -v 传参（把 shell 变量带进 awk）
awk -v threshold=100 '$3 > threshold' file

# 只看第一行（示例：表头）
awk 'NR==1{print; exit}' file

# 打印最后一列
awk '{print $NF}' file
```

## Part 10 — 全语法与工程细则补全（让它真正“可当手册”）

### 1) 命令行接口：`-F`、`-v`、`-f` 的工程意义

- **`-F re`**：设置 `FS`（字段分隔符），`re` 通常是正则。等价心智模型：在读第一条记录前就执行 `FS=re`。
- **`-v name=value`**：在 `BEGIN` 之前完成变量初始化；用于把 shell 变量/配置传进 awk（比在脚本里自己解析更可靠）。
- **`-f script.awk`**：从文件读程序（建议写复杂脚本时用，便于调试与版本管理）。

最常见组合：

```bash
awk -F',' -v min=100 '$3+0 > min {print $1, $3}' file.csv
```

为什么这里写 `$3+0`：把字段显式当数字，避免“看起来是数，但被当字符串比较”的坑。

### 2) Pattern 真值规则（理解 `awk '1' file` 的那一页）

pattern 可以是任意表达式，表达式判真常用规则：

- **数字 0** 为假，**非 0** 为真
- **空字符串**为假，**非空字符串**为真

因此：

```bash
awk '1' file   # 永真 pattern + 缺省 action => 打印所有行
```

### 3) `/re/`、`~`、`!~`：正则匹配的“低层等价式”

把这三条当作“等价变换”会极大减少困惑：

- `/re/` 等价于：`$0 ~ /re/`
- `x ~ /re/`：对 `x` 做正则匹配（比 `/re/` 更精确，因为你可指定字段/变量）
- `x !~ /re/`：不匹配

例：只在第 2 列匹配 ERROR 时输出：

```bash
awk '$2 ~ /ERROR/ {print}' file
```

### 4) 规则列表是顺序执行的：同一条记录可命中多条规则

AWK 不是“找到一个匹配就停”。对每条记录：

- 所有规则按顺序测试
- 命中的规则按顺序执行

最小示例（同一行打印两次）：

```bash
awk '
  /foo/ {print "A", NR}
  /foo/ {print "B", NR}
' file
```

如果你想命中后就不再处理该行，用 `next`：

```bash
awk '
  /foo/ {print "hit"; next}
  {print "other"}
' file
```

### 5) 语句与输出：逗号 vs 拼接（最常见的输出细节坑）

- `print $1, $2`：中间输出 `OFS`（默认空格）
- `print $1 $2`：字符串**拼接**（中间不加分隔）

这能解释为什么你明明“打印两列”，却得到粘连输出。

### 6) 表达式与运算符（AWK 当语言用的核心）

常用运算符速查：

- **算术**：`+ - * / %`
- **自增自减**：`++ --`（前置/后置均可）
- **赋值**：`=` 以及 `+= -= *= /= %=`
- **比较**：`== != < <= > >=`
- **逻辑**：`&& || !`
- **三目**：`cond ? a : b`
- **匹配**：`~ !~`
- **数组成员**：`(k in arr)`
- **字符串拼接**：把表达式直接并排写（`s = $1 "-" $2`）

### 7) 数字 vs 字符串：让比较行为“可控”

AWK 会在数字/字符串语义之间自动转换。工程上你要会“强制类型视角”：

- **强制当数字**：`x = $3 + 0`
- **强制当字符串**：`x = "" $3`

例：数值过滤更稳：

```bash
awk '{x=$3+0; if (x>100) print $0}' file
```

### 8) 更多内建变量（大脚本会频繁用到）

- **`FILENAME`**：当前输入文件名
- **`ARGC` / `ARGV`**：命令行参数数组（含输入文件名）
- **`ENVIRON`**：环境变量映射
- **`RSTART` / `RLENGTH`**：`match()` 的命中位置与长度
- **`SUBSEP`**：多维数组下标分隔符（`a[i,j]` 的底层机制）
- **`OFMT` / `CONVFMT`**：数字转字符串格式（输出一致性与精度相关）

### 9) 更多内建函数（实战更常用的一组）

#### 正则/替换（日志清洗与字段提取的核心）

- `sub(re, repl[, s])`：替换第一个匹配（默认作用于 `$0`）
- `gsub(re, repl[, s])`：替换所有匹配
- `match(s, re)`：查找并设置 `RSTART/RLENGTH`

例：把所有连续空白压成一个空格：

```bash
awk '{gsub(/[[:space:]]+/, " "); print}' file
```

#### 大小写与格式化

- `tolower(s)` / `toupper(s)`
- `sprintf(fmt, ...)`：返回格式化字符串（常用来拼 key 或构造复杂输出）

例：小写化第一列：

```bash
awk '{print tolower($1)}' file
```

### 10) 数组补全：`delete` 与“多维数组”

- 删除元素：`delete arr[k]`

为什么重要：长流处理、窗口统计、或遇到爆量 key 时需要释放状态。

“多维数组”示意（本质是把多个下标拼成一个字符串键）：

```bash
awk '
  { pair[$1,$2]++ }
  END { for (k in pair) print k, pair[k] }
' file
```

### 11) I/O 重定向与 `getline`（强大但要敬畏）

#### 输出重定向

- `print ... > "out.txt"` 覆盖写
- `print ... >> "out.txt"` 追加写
- `print ... | "cmd"` 管道到外部命令

工程提示：大量不同目标文件/命令时，配合 `close()` 避免句柄耗尽。

#### `getline` 常见形式

- `getline`：读下一条记录到 `$0`（会改变你对主循环的直觉）
- `getline x`：读到变量 `x`
- `"cmd" | getline x`：从命令输出读
- `getline x < "file"`：从文件读

调试建议：一旦用 `getline`，就更频繁打印 `NR/FNR/FILENAME` 来确认你“读到哪了”。

### 12) 可移植性与实现差异（工程上别踩雷）

常见实现：`gawk`、`mawk`、`nawk`、BusyBox `awk`。大体兼容 POSIX awk，但扩展功能/边界行为不同。

- **要可移植**：尽量写 POSIX awk 子集，少依赖实现扩展。
- **要强功能**：明确依赖 `gawk`，并在脚本/文档里写清楚（尤其是生产环境）。

### 13) 调试工作流升级：one-liner → 脚本文件

当你开始纠结引号、转义、分号、或需要多处 debug 输出时，直接改用脚本文件：

```bash
cat > script.awk <<'AWK'
BEGIN { OFS="\t" }
{ print NR, NF, $0 }
END { print "done" > "/dev/stderr" }
AWK

awk -f script.awk file
```

为什么这一步价值极高：语句边界更清晰、调试更容易、可复用与可版本化。
