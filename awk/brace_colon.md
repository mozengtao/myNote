# AWK 中 `{}` 与 `;` 的使用规则、最佳实践及心智模式

> **一句话理解：**
>
> **Pattern 决定"什么时候执行"，Action(`{}`)决定"执行什么"，`;`用于分隔同一级别的多个语句。**
>
> 永远把 AWK 当成一种**事件驱动(Event Driven)的数据处理语言**来看，而不是普通 Shell 命令。

---

# 一、AWK 程序的整体结构

AWK 程序实际上由很多条 Rule（规则）组成：

```
pattern { action }

pattern { action }

pattern { action }
```

可以理解成：

```
读取一行
      │
      ▼
判断 Rule1 是否匹配
      │
      ▼
执行 Action1

判断 Rule2 是否匹配
      │
      ▼
执行 Action2

......

读取下一行
```

ASCII Diagram：

```
                每读取一行
                     │
                     ▼
          +------------------+
          |   Rule 1         |
          | pattern { ... }  |
          +------------------+
                     │
          匹配? -----Yes------► 执行
                     │
                     ▼
          +------------------+
          |   Rule 2         |
          +------------------+
                     │
                     ▼
                ...
                     │
                     ▼
             下一条输入记录
```

因此：

```
pattern { action }
```

不是：

> 如果

而是：

> 当满足 pattern 时，就执行 action。

这是 AWK 最重要的心智模型。

---

# 二、大括号 `{}` 的作用

## 作用一：定义 Action

例如：

```awk
$3 > 100 {
    print $1
}
```

意思：

```
Pattern:
    $3 > 100

Action:
    print $1
```

等价于：

```
if ($3 > 100)
    print $1
```

但是 AWK 内部不是 if，而是 Rule。

---

## 作用二：BEGIN

```
BEGIN {
    FS=","
    OFS="\t"
}
```

表示：

```
输入开始之前执行一次
```

---

## 作用三：END

```
END {
    print total
}
```

表示：

```
所有输入结束之后执行一次
```

---

## 作用四：复杂代码块

```
{
    total++

    if ($3 > max)
        max=$3

    print $0
}
```

一个 Action 可以包含很多语句。

---

# 三、什么时候可以省略 `{}`？

如果只有默认 Action：

```
awk '/ERROR/'
```

实际上等价：

```
/ERROR/ {
    print $0
}
```

默认 Action：

```
print $0
```

例如：

```
awk '/root/'
```

输出所有匹配 root 的行。

---

如果没有 Pattern：

```
{
    print
}
```

表示：

```
每一行都执行
```

例如：

```
awk '{print $1}'
```

实际上 Pattern 为：

```
true
```

即：

```
true {
    print $1
}
```

---

# 四、分号 `;` 的作用

分号只是：

> **语句分隔符**

例如：

```
{
    a++
    print a
}
```

与

```
{ a++; print a }
```

完全一样。

---

## 示例一：同一行多个语句

```
awk '{sum+=$1; count++}'
```

相当于

```
awk '
{
    sum += $1
    count++
}
'
```

---

## 示例二：BEGIN

```
BEGIN { FS=":"; OFS="," }
```

等价：

```
BEGIN {
    FS=":"
    OFS=","
}
```

---

## 示例三：多个 Rule

```
/ERROR/ { err++ }
/WARN/  { warn++ }
END      { print err,warn }
```

也可以写成：

```
/ERROR/ { err++ };
/WARN/  { warn++ };
END { print err,warn }
```

Rule 之间的分号不是必须。

通常不用。

---

# 五、什么时候必须写分号？

## 情况一：一行多个语句

```
{ a++; b++; print a }
```

必须：

```
{ a++; b++; }
```

不能：

```
{ a++
b++ }
```

因为已经换行了。

---

## 情况二：BEGIN 中

```
BEGIN { FS=":"; OFS="\t"; count=0 }
```

---

## 情况三：for

```
for(i=1;i<=NF;i++)
```

这里三个部分必须用分号。

```
for(initialization;
    condition;
    increment)
```

---

## 情况四：do while

```
do {
    ...
} while(...)
```

这里与 C 一样。

---

# 六、不需要分号的时候

例如：

```
{
    print
    total++
}
```

换行即可。

推荐这种。

---

# 七、大括号最佳实践

推荐：

```
{
    statement1
    statement2

    if (...)
        ...

    print ...
}
```

不要：

```
{statement1;statement2;if(...)print ...}
```

后者可读性极差。

---

# 八、复杂 AWK 的推荐模板

## 推荐模板

```awk
#!/usr/bin/awk -f

##############################################################################
# BEGIN
##############################################################################

BEGIN {

    FS = ","
    OFS = "\t"

    total = 0
    matched = 0
}

##############################################################################
# Skip empty line
##############################################################################

NF == 0 {
    next
}

##############################################################################
# Skip comment
##############################################################################

$1 ~ /^#/ {
    next
}

##############################################################################
# Validation
##############################################################################

NF < 4 {

    printf("Invalid record: %s\n", $0) > "/dev/stderr"

    next
}

##############################################################################
# Business Rule
##############################################################################

$3 > 80 {

    matched++

    score_sum += $3

    users[$1] = $3
}

##############################################################################
# Statistics
##############################################################################

{

    total++
}

##############################################################################
# END
##############################################################################

END {

    print "========== Summary =========="

    print "Total Records :", total
    print "Matched       :", matched

    if (matched)
        print "Average Score :", score_sum / matched

    print ""

    print "Matched Users"

    for (u in users)
        printf("%-20s %d\n", u, users[u])
}
```

这种布局非常适合维护大型 AWK 脚本。

# 九、复杂脚本推荐布局

```
BEGIN

↓

Input Filter

↓

Validation

↓

Business Logic

↓

Statistics

↓

END
```

对应：

```
BEGIN

↓

初始化

↓

过滤垃圾数据

↓

检查格式

↓

真正业务

↓

统计

↓

输出结果
```

这是很多大型 AWK 脚本采用的组织方式。

# 十、关于 `{}` 的心智模型

不要把 `{}` 看成 C 的代码块。

而应该看成：

```
事件处理函数(Event Handler)
```

例如：

```
/ERROR/ {

}
```

理解成：

```
当收到 ERROR 事件时：

执行下面代码
```

```
BEGIN {

}
```

理解成：

```
程序启动事件
```

```
END {

}
```

理解成：

```
程序退出事件
```

```
$3 > 100 {

}
```

理解成：

```
当某条记录满足条件：

执行下面代码
```

因此：

```
Pattern  --->  Event

Action   --->  Event Handler
```

---

# 十一、关于 `;` 的心智模型

很多人认为：

```
;
```

是 AWK 特殊语法。

其实不是。

它只是：

> **同一级语句之间的分隔符（Statement Separator）**

因此：

```
a=1
b=2
print a+b
```

与

```
a=1; b=2; print a+b
```

完全一致。

推荐原则：

- **一条语句一行**：不写 `;`，可读性最佳。
- **单行快速命令**：使用 `;` 分隔多个语句。
- **`for(init; cond; step)`**：三个部分必须使用 `;`。
- **Rule 之间**：通常换行即可，不必使用 `;`。

---

# 十二、AWK 编写黄金法则（Golden Rules）

| 原则 | 推荐做法 | 原因 |
|------|----------|------|
| 一个 Rule 做一件事 | 按过滤、校验、统计、输出拆分 Rule | 易维护、符合事件驱动思维 |
| `BEGIN` 只做初始化 | 设置 `FS`、`OFS`、变量 | 避免混入业务逻辑 |
| `END` 只做汇总输出 | 统一打印统计信息 | 保持流程清晰 |
| 一行一个语句 | 用换行代替 `;` | 可读性更高 |
| Action 使用多行 `{}` | 即使只有几条语句也保持缩进 | 后续扩展方便 |
| 善用 `next` | 尽早过滤无效数据 | 减少嵌套，提高效率 |
| Rule 按处理阶段排序 | BEGIN → Filter → Validate → Business → Statistics → END | 形成稳定的处理流水线 |

---

# 十三、最终心智模型（Architecture）

```
                AWK Program

        ┌──────────────────────┐
        │ BEGIN                │
        │ 初始化                │
        └──────────┬───────────┘
                   │
                   ▼
      每读取一条记录（Record）
                   │
                   ▼
      ┌─────────────────────────────┐
      │ Rule 1                      │
      │ Pattern { Action }          │
      └─────────────────────────────┘
                   │
      ┌─────────────────────────────┐
      │ Rule 2                      │
      └─────────────────────────────┘
                   │
      ┌─────────────────────────────┐
      │ Rule 3                      │
      └─────────────────────────────┘
                   │
                   ▼
      下一条记录
                   │
                   ▼
        ┌──────────────────────┐
        │ END                  │
        │ 汇总输出              │
        └──────────────────────┘


每个 Rule：

Pattern
    │
    ▼
是否匹配？
    │
 Yes
    ▼
{
    Statement1
    Statement2
    Statement3
}
```

> **核心心智模式：**
>
> 把 AWK 看成一个**按记录（Record）驱动的事件处理器**：`Pattern` 决定事件何时触发，`{ Action }` 是事件处理函数，`;` 只是同一级语句的分隔符。按照 **BEGIN → Filter → Validate → Business → Statistics → END** 的流水线组织脚本，可以写出结构清晰、易维护的复杂 AWK 程序。
